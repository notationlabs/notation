Problem: the AWS router DSL is doing two jobs (route DSL + AWS wiring), so it could not be reused by another provider without copy/paste. I chose an adapter seam because it keeps the DSL provider-agnostic and keeps AWS-specific wiring in AWS. Grounding: commit d3c068d adds packages/serverless-router/src/index.ts and rewires packages/aws/src/api-gateway/router.ts to call the core router via an adapter, while packages/aws/src/api-gateway/auth.ts re-exports the shared auth types. I also had to add an explicit return type for the AWS router to fix a tsup DTS portability error in router.ts.
